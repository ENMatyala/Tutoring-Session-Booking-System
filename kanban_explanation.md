### Kanban Board Explanation for Tutor Booking System

---

#### 1. Definition of a Kanban Board

A **Kanban board** is a visual project management tool used to track tasks as they move through different stages of work. It helps teams organize, monitor, and manage their workflow efficiently.

The board is made up of **columns**, where each column represents a stage in the development process, such as:
- Backlog  
- Todo  
- In Progress  
- Testing  
- Done  

Tasks are displayed as **cards** and move across these columns as work progresses.

---

#### 2. How the Kanban Board Visualizes Workflow

In the Tutor Booking System project, the Kanban board is used to represent the flow of tasks from planning to completion.

- The **Backlog** column contains all user stories and tasks from Assignment 6.
- The **Todo** column includes tasks selected for the current sprint.
- The **In Progress** column shows tasks that are actively being worked on.
- The **Testing** column is used to verify completed features before final approval.
- The **Done** column contains completed and verified tasks.

This structure makes it easy to see:
- What work still needs to be done  
- What is currently in progress  
- What has already been completed  

---

#### 3. Work-In-Progress (WIP) Limits

To prevent too many tasks from being worked on at the same time, the Kanban board uses **Work-In-Progress (WIP) limits**.

For this project:
- A maximum of **3 tasks** are allowed in the **In Progress** column at any time.

This helps to:
- Maintain focus on completing tasks  
- Reduce multitasking  
- Avoid bottlenecks in the workflow  

---

#### 4. Support for Agile Principles

The Kanban board supports Agile development in several ways:

##### Continuous Delivery
Tasks move continuously through the board without waiting for fixed sprint deadlines. This allows features like tutor search and booking to be delivered incrementally.

##### Flexibility and Adaptability
New tasks or changes can be added to the **Backlog** at any time without disrupting ongoing work.

##### Transparency
All tasks and their statuses are visible to everyone, making it easier to track progress and identify delays.

##### Collaboration
Team members can clearly see responsibilities and task ownership, improving coordination and communication.

---

#### 5. Practical Example in This Project

For example:
- Task **T1 (Develop user registration API)** starts in **Todo**, moves to **In Progress**, then to **Testing**, and finally to **Done** once completed.
- If a task cannot proceed due to a dependency, it is moved to the **Blocked** column until the issue is resolved.

This ensures that every task follows a clear and structured workflow.

---

#### 6. Conclusion

The Kanban board is an effective tool for managing the Tutor Booking System project. It provides clear visibility of tasks, improves workflow efficiency, and supports Agile principles such as continuous delivery and adaptability.

By using this approach, the project team can ensure that tasks are completed efficiently while maintaining high quality and organization.
