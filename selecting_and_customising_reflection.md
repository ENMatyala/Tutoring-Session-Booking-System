### Reflection on GitHub Project Templates and Kanban Implementation

---

#### 1. Challenges in Selecting a Template

One of the main challenges in this assignment was choosing the most appropriate GitHub Project template for the Tutor Booking System. GitHub provides multiple templates, each designed for different purposes, such as basic task tracking, bug management, or team planning.

Initially, templates like **Basic Kanban** and **Automated Kanban** seemed suitable due to their simplicity. However, they lacked important Agile features such as backlog management and structured workflow planning. This made it difficult to directly align them with the sprint backlog and user stories created in Assignment 6.

Another challenge was understanding how each template supports Agile practices. While some templates offer automation, they do not always support sprint-based workflows or prioritization, which are essential for managing a structured project like the Tutor Booking System.

After comparing the templates, the **Team Planning** template was selected because it provides a balance between structure and flexibility, making it more suitable for Agile development.

---

#### 2. Challenges in Customizing the Kanban Board

Customizing the Kanban board to meet the assignment requirements also presented some challenges.

One challenge was deciding which additional columns to include. While the default template provides basic workflow stages, it was necessary to extend the board by adding columns such as **Testing** and **Blocked**. This required careful consideration to ensure that the workflow remained clear and not overly complex.

Another challenge was mapping the sprint tasks (T1–T12) from Assignment 6 into the Kanban board. Each task needed to be properly represented as an issue and placed in the correct column. Ensuring that tasks were correctly labeled and assigned also required attention to detail.

Additionally, linking issues and maintaining a consistent workflow required understanding how GitHub Projects integrates with issues, labels, and milestones. This was initially confusing but became clearer with practice.

---

#### 3. Comparison with Other Tools

GitHub Projects was compared with other popular project management tools such as **Trello** and **Jira**.

- **Trello** is known for its simplicity and ease of use. It provides a clean drag-and-drop interface, making it ideal for beginners or small projects. However, it lacks advanced features such as deep integration with code repositories and automated workflows.

- **Jira** is a powerful tool widely used in industry for Agile project management. It supports advanced features such as sprint planning, reporting, and detailed workflow customization. However, it can be complex and overwhelming, especially for smaller projects or beginners.

- **GitHub Projects**, on the other hand, offers a balance between these two tools. It integrates directly with the repository, allowing seamless linking of issues, pull requests, and code changes. While it may not be as feature-rich as Jira, it provides enough functionality to effectively manage Agile workflows within a development environment.

---

#### 4. Lessons Learned

This assignment provided valuable insights into Agile project management and the use of Kanban boards.

It highlighted the importance of:
- Selecting the right tools based on project needs  
- Structuring workflows clearly to improve efficiency  
- Using visual tools to track progress and identify bottlenecks  

Additionally, working with GitHub Projects improved my understanding of how development tasks, issues, and workflows are connected in a real-world environment.

---

#### 5. Conclusion

Overall, this assignment demonstrated how GitHub Project templates and Kanban boards can be used to effectively manage a software development project. Despite initial challenges in selecting and customizing the template, the final solution provides a clear, organized, and efficient workflow for the Tutor Booking System.
