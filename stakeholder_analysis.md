# Stakeholder Analysis – Tutor Booking System

## Stakeholders Overview

### 1. Student
- **Role**: Uses the system to find and book tutors.
- **Key Concerns**: Easy booking process, tutor availability, affordability.
- **Pain Points**: Difficulty finding qualified tutors, scheduling conflicts.
- **Success Metrics**: Booking completed in <2 minutes; 90% successful session attendance.

### 2. Tutor
- **Role**: Provides tutoring services and manages availability.
- **Key Concerns**: Fair exposure to students, flexible scheduling, timely payments.
- **Pain Points**: Missed bookings, unclear schedules, late payments.
- **Success Metrics**: 80% schedule utilization; payments processed within 24 hours.

### 3. Platform Administrator
- **Role**: Manages system operations, users, and configurations.
- **Key Concerns**: System reliability, user satisfaction, data accuracy.
- **Pain Points**: Manual user management, handling disputes.
- **Success Metrics**: 99% uptime; reduction in support tickets.

### 4. Payment Processor / Finance Admin
- **Role**: Handles transactions and financial records.
- **Key Concerns**: Secure transactions, accurate billing.
- **Pain Points**: Failed or delayed transactions.
- **Success Metrics**: 99.9% successful transactions; zero financial discrepancies.

### 5. IT / DevOps Team
- **Role**: Maintains infrastructure and deployment.
- **Key Concerns**: System uptime, scalability, performance.
- **Pain Points**: Server downtime, scaling issues.
- **Success Metrics**: 99.9% uptime; response time <2 seconds.

### 6. Support Staff
- **Role**: Assists users with issues and queries.
- **Key Concerns**: Quick issue resolution, user satisfaction.
- **Pain Points**: Repetitive queries, lack of system insights.
- **Success Metrics**: Resolve 90% of tickets within 24 hours.
